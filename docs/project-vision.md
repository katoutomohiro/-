## 3つのAIエージェントによる開発体制

本プロジェクトは、3つのAIエージェントが役割分担を明確にして協働することで、開発効率を最大化します。

### 1. v0（戦略的設計者・アーキテクト）

**接続場所**: Web（v0.app）

**役割**:
- プロジェクト全体のアーキテクチャ設計
- 新機能の要件定義と実装計画
- UI/UXデザインの提案と生成
- 複数ファイルにまたがる大規模なコード生成
- プロジェクトビジョンの管理
- 技術的な意思決定のサポート
- デザインインスピレーションの生成

**担当フェーズ**:
- 新機能の設計書作成
- コンポーネント・ページの生成
- データ構造の設計
- 開発ロードマップの管理
- GitHub Copilotへの詳細実行指示の作成

### 2. Codex CLI（ファイル操作の実行者）

**接続場所**: ターミナル（CLI経由）

**役割**:
- リポジトリ内のファイル操作（参照・編集）
- コマンド実行（`pnpm`, `git`, `tsc`等）
- リファクタの提案
- ファイルの読み書き（CLI経由）
- 相談ベースでの作業

**担当フェーズ**:
- v0が生成したコードをファイルに書き込む
- `pnpm dev`, `pnpm build`, `pnpm tsc --noEmit`の実行
- `git add`, `git commit`, `git push`の実行
- ファイルの内容確認・編集
- データベーススクリプトの実行
- パッケージの追加・更新

**制約事項**:
- 自動マージは行わない
- GitHub上での操作は相談ベース
- リモート影響のある作業は相談ベース

### 3. VS Code内エージェント（インライン支援の専門家）

**接続場所**: VS Code エディタ内（GPT-5 mini等）

**役割**:
- ファイル内容を直接読み取り
- リアルタイムの提案と自動修正
- IDE内でのインライン支援
- コード補完
- 即時的なフィードバック

**担当フェーズ**:
- エディタ内でのコード補完
- タイプ中のリアルタイム提案
- 小規模な修正の即時実行
- コードの最適化提案
- ドキュメントコメントの追加

## 推奨ワークフロー

### パターン1: 新機能開発

\`\`\`

   ↓ 設計書とコード生成
   ↓ GitHub Copilot実行指示を作成

   ↓ v0の指示に従ってファイルを作成・編集

3. VS Code内エージェント

   ↓ エディタ内で即時修正
   
2. Codex CLI
   ↓ pnpm tsc --noEmit でエラーチェック
   ↓ git commit → git push
\`\`\`

### パターン3: バグ修正（大規模）

\`\`\`
1. v0
   ↓ 問題の分析と修正計画
   ↓ 修正コード生成
   
2. Codex CLI
   ↓ ファイルを編集
   ↓ エラーチェック・動作確認
   
3. VS Code内エージェント
   ↓ 細部の調整
   
4. Codex CLI
   ↓ git commit → git push
\`\`\`

### パターン4: データベース操作

\`\`\`
1. v0
   ↓ SQLスクリプト生成（scripts/フォルダ）
   
2. Codex CLI
   ↓ スクリプトをファイルに保存
   ↓ スクリプト実行
   ↓ git commit → git push
\`\`\`

## 衝突回避のルール

3つのエージェントが同じファイルを編集する可能性があるため、以下のルールを守ります。

1. **v0が大規模な変更を生成した場合**:
   - Codex CLIがファイルに書き込む前に、VS Code内エージェントの編集を保存
   - Codex CLIが書き込み後、VS Code内エージェントが微調整

2. **VS Code内エージェントが編集中の場合**:
   - Codex CLIは編集が完了するまで待機
   - 相談ベースで進める

3. **Codex CLIがコマンド実行中の場合**:
   - 他のエージェントは実行完了まで待機

4. **Git操作は必ずCodex CLIが実行**:
   - v0とVS Code内エージェントはGit操作を行わない
